xilli linux

xilli linux is a pragmatic linux distribution for everyday use. xilli is pronounce like the english word chilly Designed with love, xilli linux removes many of the outdated technologies from the typical linux stack. Our development objectives for xilli are that it be: * simple * understandable * manageable * and command-line friendly

Papers & Sources

As xilli is currently under heavy development, it is not recommended for daily use yet. If you want to experiment with getting it running on your system you can get all of the sources from github.

To checkout the source from git, the current public git mirror is: git clone https://github.com/cthulhuology/xilli.git

The README file has the most current instructions on how to complie the base system. Currently it is know to compile on Debian Jessie and SteamOS brewmaster.

xilli concepts

There are many aspects of xilli linux that are experimental, and as such some of the concepts presented are different from other linuxes you may be accustomed to. First off, xilli doesn't follow the FHS. Instead it uses the following:

The key observations is that from a 3rd party software perspective, /, /usr/, and /usr/local/ are equivalent. Also, /home is root's home directory (xilli is single user). The source code to the system is contained in /src, and all of the build tools are in /tools.

Xilli eschews package management software. Rather to install software on xilly one types: /install/package-name

This will build from source from the /src/ directory the package int the /tmp/ directory, and install it into /

Eventually, xilli will support: /uninstall/package-name

But that is currently a rather low priority, as many useful programs are still not built for xilli, and much of the /src directory still doesn't have build scripts. All of the build instructions are just bash scripts, and are easily read, modified, and understood by anyone.
